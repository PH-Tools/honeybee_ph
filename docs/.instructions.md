# docs/ — PH-Tools Hub Spoke Convention

This folder is a **spoke** in the PH-Tools documentation hub ([ph-docs](https://github.com/PH-Tools/ph-docs)). The hub sparse-clones this folder into its build, so the structure here must follow the convention exactly.

## Required Files

| File | Purpose |
|------|---------|
| `index.md` | Landing page for this library's section on the docs site |
| `nav.yml` | Navigation subtree — merged into the hub's `mkdocs.yml` automatically |

## nav.yml Format

A single top-level `nav:` key containing a list of entries. Paths are **relative to this `docs/` folder**. The hub prefixes all paths with the library's `id` (from `libraries.yml`) during assembly — do not add prefixes yourself.

```yaml
nav:
  - Overview: index.md
  - Reference:
    - Some Page: reference/some-page.md
```

Nested groups become nav sections on the built site.

## Optional Folder Structure

```
docs/
├── index.md          # Required
├── nav.yml           # Required
├── user/             # Tutorials, how-to guides
├── dev/              # API docs, architecture
└── reference/        # Schema maps, field guides
```

Organize however makes sense — the hub reads `nav.yml`, not folder names.

## How the Hub Build Works

1. `fetch_spokes.py` sparse-clones `docs/` → `assembled/{lib_id}/`
2. `build_nav.py` reads `nav.yml`, prefixes paths with `{lib_id}/`, injects into `mkdocs.yml`
3. MkDocs builds from `assembled/`

## Rules

- **Every `.md` file must be listed in `nav.yml`** — unlisted files won't appear in navigation
- **No cross-repo relative links** — link to other libraries via absolute site URLs (`/honeybee-ph/...`), not `../`
- **Markdown only** — the hub uses MkDocs Material; standard Markdown + admonitions + the hub's CSS components (`.ph-meta-strip`, `.ph-feature-grid`, etc.) are available
- **Images/assets** — put in a local `assets/` subfolder; reference with relative paths
- **Don't duplicate hub content** — if something belongs to the hub (e.g., cross-cutting guides), it stays there
