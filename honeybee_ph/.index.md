# honeybee_ph/ — core PH data model

The core package: Passive House attributes attached to base Honeybee objects (`Room`, `Model`, etc.). Registered onto Honeybee via `_extend_honeybee_ph.py`. See `../context/ARCHITECTURE.md`.

## Key modules

- `space.py` — interior PH spaces (iCFA / TFA), the central spatial object.
- `bldg_segment.py` — building segment grouping.
- `phi.py`, `phius.py` — PHI and Phius certification data/thresholds.
- `site.py`, `team.py`, `foundations.py` — project site, team, and foundation objects.
- `_base.py` — shared base class.
- `_extend_honeybee_ph.py` — registers `.properties.ph` onto Honeybee objects (runs on import).

## Subfolders

- `properties/` — per-host-object PH property classes (own the `to_dict()`/`from_dict()` for each host type).
- `cli/` — command-line entry points.

## Notes
- Tests: `../tests/test_honeybee_ph/`. Every model class round-trips through HBJSON — see `../context/CODING_STANDARDS.md` §Serialization.
